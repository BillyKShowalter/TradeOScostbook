# TradeOS Construction Knowledge Engine

The TradeOS Construction Knowledge Engine is a structured, machine-readable repository of construction costing data, templates, safety instructions, and logic rules. It serves as the "brain" of TradeOS's estimating systems, capturing contractor expertise and preparing it for automated processing.

---

## 1. What the Knowledge Engine Is
This repository is a localized knowledge sandbox containing:
* **Costbook Databases**: 1,795 pre-priced contractor cost items across 24 trades, and 40 assemblies.
* **Logic Rules**: Rules for data validation, unit casing, pricing sanity thresholds, and fuzzy deduplication.
* **Canonical JSON Schemas**: Validated structures to ensure database migrations conform to strict schemas.
* **AI Prompts & Skills**: Core instruction cards for specialized autonomous coding and database agents.

---

## 2. How it Differs from the TradeOS Application
* **The TradeOS App**: A cloud-hosted enterprise SaaS product running on web/mobile platforms that allows contractors to write estimates, send invoices, and manage client communications in real time.
* **The Knowledge Engine**: A local repository housing structured data, rules, and Python pipelines. It does **not** contain user interfaces or desktop apps. Instead, it compiles, refines, and formats cost data. Its outputs (`exports/json/costbook.json` and `exports/sql/sync_final.sql`) are deployed directly to the backend database of the TradeOS app.

---

## 3. How AI Systems Consume It
The Knowledge Engine is designed from the ground up for LLM-based agentic workflows:
1. **Agent Prompts**: Specialized LLM agents (such as the Costbook Architect or Assembly Builder) read the prompts in `prompts/agents/` and skills in `.agent/skills/` to generate or update costbook components.
2. **Retrieval-Augmented Generation (RAG)**: AI estimators, proposal generators, and voice systems query the schemas in `schemas/` and the rules/reasoning guidelines in `knowledge/` to perform accurate parameter-based assembly matches and pricing.
3. **Deterministic Verification**: Any content generated by AI agents must pass through the validation rules in `knowledge/validation-rules/` and the pricing sanity checks in `knowledge/pricing-sanity/` before being exported, ensuring complete safety.
