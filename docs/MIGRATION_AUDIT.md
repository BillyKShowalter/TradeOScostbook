# Migration Audit

Full findings from the 2026-07-03 audit of the legacy Costbook/Swift/Knowledge-Engine migration and the broader repository documentation. Covers: duplicate-migration verification, stale-documentation findings, and what was corrected vs. what's left as a recommendation.

## 1. Duplicate-migration check

**No duplicate knowledge-engine payload exists in the repository.**

- `packages/knowledge-engine/` (the full payload: `knowledge/`, `schemas/`, `scripts/`, `runtime/`, `review/`, `exports/`, `prompts/`, `pipelines/`, `docs/`, `agent-skills/`, `legacy-archive/`) was committed once, in `e31c1c8`.
- `packages/legacy-tradeos-reference/` was about to receive a second full copy of the same payload in a later session, but a `diff -rq` comparison against `packages/knowledge-engine/` showed the two were byte-for-byte identical before the second copy was committed. Only the 5 transfer-planning docs were kept there; the redundant payload copy was never committed.
- No other `TRANSFER_TO_TRADEOS*`-named directories, and no leftover temporary migration folders, exist anywhere else in the repository (confirmed via a repo-wide search for the transfer folder's original name and common temp-migration naming patterns).

**Distinct, smaller-scale duplication does exist *inside* `packages/knowledge-engine/`**: the same `costbook.json` cost-item data is present in at least 7 locations at different generations (`exports/json/`, `knowledge/knowledge/cost-items/`, `pipelines/exports/json/`, `pipelines/knowledge/cost-items/`, and three copies under `legacy-archive/archive/legacy-sync-experiments/`). This is internal redundancy from the original knowledge-engine's own iterative workflow, not a symptom of the transfer into this repo. See `KNOWLEDGE_ENGINE_STATUS.md` for the full list and a recommendation.

## 3. What was verified as still accurate (not touched)

- Root `README.md`'s prose describing `app/` and `web/` (aside from the missing `packages/` mention) was accurate and already used `backend/` correctly.
- `docs/design/` (3 PNG screenshots) matches the admin-shell fidelity work narrated in `end-of-session-note.md` — no markdown to check, no staleness found.
- `docs/tradeos-mvp-foundation-plan.md` is explicitly foundational/historical; its auth-endpoint naming guesses and "next pass: OpenAI" note are superseded by newer docs but the doc itself doesn't claim otherwise, so it wasn't flagged as broken, just superseded.
- `app/.env.example` and `app/prisma/schema.prisma`/migrations were spot-checked against `app/README.md`'s claims and found consistent (aside from the fixed staleness above).
